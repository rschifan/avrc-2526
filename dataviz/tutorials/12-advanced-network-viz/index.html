DV12 — Advanced network visualization

What you should already know

This lecture builds on L11 (force layouts, drag, hover, zoom on the GoT network) and assumes comfort with the data join, scales, and SVG primitives from L9 / L10.

Tutorials

Most use the same Game-of-Thrones co-occurrence network as the running example. The first four introduce alternatives to the node-link view; then we take networks into the time domain and beyond D3, and the capstone (12g) closes the lecture by putting four views of the same data side by side.

  1. 12a — Adjacency matrix
    Building an N × N matrix from an edge list; cell-per-link join; why row ordering (alphabetical / community / centrality / barycenter) is what determines whether community blocks become visible; row + column highlight on hover.
  2. 12b — Chord diagram
    d3.chord() + d3.ribbon() + d3.arc() on the same matrix; community ordering on the rim; hover-fade to isolate a node's flow; when chord works (and doesn't).
  3. 12c — Sankey diagram
    Visualizing flows on a directed acyclic graph; the d3-sankey plugin; nodeAlign, nodePadding, extent; reading width = quantity; and a linked Sankey + force view driven by one synchronized hover.
  4. 12d — Hierarchical edge bundling
    Wrapping a flat node list in a synthetic hierarchy; d3.cluster on a circle; d3.lineRadial with curveBundle.beta(0.85); hover to colour a character's edges by the community they reach. Plus the force-directed family (FDEB), which needs no hierarchy at all.
  5. 12e — Dynamic and streaming graphs
    What happens when the graph changes over time. The three sub-patterns (streaming / snapshots / continuous); selection.join with key functions for stable transitions; rebuilding forceLink after every update; gentle reheats with alpha(0.4). Streaming demo (one node every 600 ms) and snapshot demo (three time slices with position-preserving transitions).
  6. 12f — Beyond D3: the network viz ecosystem
    Where D3 stops being the right answer. Cytoscape.js, sigma.js + Graphology, Cosmograph, Gephi, vis-network, PyVis — what each one is for and where its ceiling sits. The architectural split between graph data structure and renderer, and when to stay in D3.
  7. 12g — Capstone: same data, four views
    Force / matrix / chord / bundling on the same 30 GoT characters and the same colour scale, wired to a single synchronized hover; a decision guide for picking the right view per question; and a closing synthesis of the whole lecture.

Accessibility (cross-cutting note)

Force-directed networks are notoriously inaccessible — SVG geometry conveys no semantics to screen readers, and Canvas / WebGL conveys none at all. The matrix view in 12a is the exception: it has a natural <table>-shaped accessible companion (one <tr> per row, one <td> per cell). When a11y matters, ship the matrix as role="presentation" SVG plus a parallel <table> with aria-hidden="true" on the visual. The chord and sankey tutorials (12b, 12c) have less natural fallbacks; ship a summary table with the totals so screen readers get the same numbers the chart shows.

Setup

Self-contained: every local asset (_shared/chart.css, _shared/chart.js, datasets/got.json) is inside this folder. Run from a local server to satisfy CORS (the JSON load fails over file://):

cd 12-advanced-network-viz
python -m http.server 8000
# open http://localhost:8000/

D3 loads from its CDN. Two tutorials add one more library from a CDN: d3-sankey (12c) and d3.ForceEdgeBundling (12d).
