12a — Ordered adjacency matrix

The top-40 Game-of-Thrones characters as an N × N matrix: one cell per edge, coloured by weight. Switch the row/column ordering and watch community blocks appear. Source tutorial →

What this example shows

  • Two encodings (see the legend). A cell's fill runs light → dark with the edge weight; the thin coloured strips on the top and left edges give each character its community colour, so the blocks are framed without cluttering the cells.
  • Ordering is the visualization. The matrix never changes — only the index permutation does. Alphabetical scatters the cells; by community and barycenter pull the dense blocks onto the diagonal.
  • Barycenter reordering moves each row next to the weighted average position of its neighbours; a few row→column passes and the blocks emerge from the data alone, with no community labels.
  • Row + column hover guides make the cross-reference effortless — the two grey strips show which pair a cell belongs to, and the matching labels go bold.
