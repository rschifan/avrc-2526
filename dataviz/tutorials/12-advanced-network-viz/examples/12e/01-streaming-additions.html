12e §4 — Streaming growth (preferential attachment)

A graph that grows one node at a time by the Barabási–Albert rule: each arrival links to M = 2 existing nodes, chosen with probability proportional to their current degree. Node size and shade track degree, so you can watch a few hubs pull away from the crowd. Source tutorial →

What this example shows

  • Emergence from a simple rule. "The rich get richer" — popular nodes attract most new links, producing the heavy-tailed degree distribution of a scale-free network. It is only visible because the graph is animated as it grows.
  • The degree-weighted pick. An endpoints pool holds every node id once per incident edge, so a uniform draw from it is automatically ∝ degree.
  • Keyed joins + gentle reheat. Nodes are joined by d.id and links by a source-target string; each update rebuilds forceLink and reheats at alpha(0.5) so survivors keep their positions.
  • Re-encoding on update. The update callback re-runs the radius transition, so hubs visibly swell as their degree climbs.
