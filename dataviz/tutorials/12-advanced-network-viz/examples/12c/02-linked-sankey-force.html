12c — Linked Sankey + force views

The same flow drawn two ways at once: a Sankey on top, a force-directed view of the same nodes below, and a single hover handler that lights up the focal node and its edges in both panels simultaneously. Source tutorial →

What this example shows

  • One dispatcher, two views. A single highlightAcross(node) toggles .focus / .faded classes on both the Sankey and the force panel, so hovering either one highlights the same character everywhere.
  • What each view gives up. The Sankey encodes stage order and width = quantity; the force view encodes neither — it only balances springs, so the clean three-stage structure collapses into a ball. That contrast is why Sankey exists for flow data.
  • Tooltip shows pre-computed in/out degree (summed edge value).
