12d — Edge bundling

Edge bundling tames a dense hairball by routing edges that travel together along shared paths, the way cables get tied into looms. This tutorial covers both families: the hierarchical kind (§1–4), which groups edges by a tree the data already implies (folders containing files, communities containing nodes), and the force-directed kind (§5, FDEB), which needs no hierarchy at all. Both turn a tangle of straight lines into a far quieter picture.

1. Build a hierarchy on top of the flat node list

The nodes in our GoT JSON are flat — they only carry a community id. To bundle, we wrap them in a synthetic two-level tree (root → community → character) and feed that to d3.hierarchy:

const root = d3.hierarchy({
    name: "got",
    children: communities.map(c => ({
        name: "c" + c,
        children: nodes.filter(n => n.community === c)
                       .map(n => ({ name: n.id, ...n }))
    }))
});

Then a circular cluster layout places every leaf on the perimeter, grouped by community:

const cluster = d3.cluster().size([2 * Math.PI, radius - 80]);
cluster(root);                             // mutates: each node gets x (angle) and y (radius)

2. Path the edges through the lowest common ancestor

For each link we look up the source and target leaves in the hierarchy and draw the curve via their least common ancestor. d3.lineRadial with a bundled curve generates the path string:

const line = d3.lineRadial()
    .curve(d3.curveBundle.beta(0.85))
    .radius(d => d.y)
    .angle(d => d.x);

paths.attr("d", l => line(l.source.path(l.target)));   // hierarchy.path(other)

The beta parameter on curveBundle controls how tightly bundled the curves are: 0 = straight lines through the LCA, 1 = pulled completely toward the LCA. Values near 0.85 are the conventional sweet spot.

4. When bundling helps

  • ✅ The graph has a meaningful tree above it (folder hierarchy, package structure, community partition) and most edges connect siblings within the same subtree.
  • ✅ You want the global picture without the visual cost of a node-link hairball.
  • ❌ The hierarchy is artificial or doesn't reflect actual edge structure — bundling forces curves through arbitrary midpoints and the picture lies.
  • ❌ Reading individual edges matters — bundles share strokes and become ambiguous.

5. The other family — force-directed edge bundling

The radial layout above uses an explicit hierarchy to define the bundles. The other canonical approach (Holten & van Wijk, 2009) needs no hierarchy at all: subdivide every edge into a chain of control points, pin the two endpoints, and let the control points of nearby, similarly-directed edges attract each other — so edges that run roughly parallel merge into a bundle. We use the d3.ForceEdgeBundling implementation: feed it node positions and an edge list, and it returns, for each edge, a poly-line of subdivision points to draw a curve through:

// 1. every node needs an (x, y) — here from a quick force pass
const nodes = {};                          // { id: {x, y}, … }
graphNodes.forEach(d => nodes[d.id] = { x: d.x, y: d.y });

// 2. edges reference node ids
const edges = links.map(l => ({ source: l.source, target: l.target }));

// 3. run FDEB — returns, per edge, an array of subdivision points pulled
//    toward compatible (parallel, nearby) edges
const bundled = d3.ForceEdgeBundling()
    .nodes(nodes)
    .edges(edges)
    .bundling_stiffness(0.1)
    .step_size(0.1)
    .compatibility_threshold(0.6)();

// 4. draw each edge as a smooth curve through its subdivision points
const line = d3.line().curve(d3.curveBasis).x(d => d.x).y(d => d.y);
edgePaths.data(bundled).join("path").attr("d", line);

Compared with the radial bundling above, force-directed bundling is easier to read at a glance but loses the obvious community grouping that the explicit hierarchy provides. Both are legitimate — pick whichever matches the structure you want to highlight.

Takeaway. Two bundling families: hierarchical (this tutorial's first demo) needs a tree; force-directed (Holten & van Wijk, second demo) needs only the original edge list. The first reveals group structure, the second emphasises pure edge-flow patterns. Both compress edge ink dramatically compared to a straight node-link view.
