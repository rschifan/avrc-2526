12b — Chord diagram

A chord diagram is the matrix of 12a bent into a circle. Nodes become arcs on the perimeter; edges become curved ribbons across the interior, with width proportional to weight. It's compact, attractive, and works best for small networks (≲ 50 nodes) where each pair's flow matters.

1. The two D3 generators

Two helpers do almost all the work:

  • d3.chord() — takes the same N × N matrix as 12a and produces angle ranges for each group (the arcs on the rim) and each chord (the ribbons inside).
  • d3.ribbon() — a path generator that turns a chord (source angle range, target angle range) into the curved SVG path that connects them.
const chord = d3.chord()
    .padAngle(0.04)                     // gap between adjacent groups
    .sortSubgroups(d3.descending);     // big sub-arcs first

const layout = chord(matrix);          // { groups: [...], …chord ribbons }

const arc    = d3.arc().innerRadius(innerR).outerRadius(outerR);
const ribbon = d3.ribbon().radius(innerR);

2. Drawing the rim and the ribbons

Two joins, one per layer:

// arcs (one per node)
g.append("g").selectAll("path")
    .data(layout.groups).join("path")
        .attr("d", arc)
        .attr("fill", d => colour(nodes[d.index].community));

// ribbons (one per chord = pair of nodes)
g.append("g").attr("fill-opacity", 0.6)
    .selectAll("path")
    .data(layout).join("path")
        .attr("d", ribbon)
        .attr("fill", d => colour(nodes[d.source.index].community));

3. Hover to isolate a node's flow

The default chord layout shows every ribbon at once — very busy. A hover handler that hides every ribbon except those touching the hovered arc (and dims the arcs it doesn't reach) turns it into a focus tool. Hide both the fill and the stroke of the unconnected ribbons, or their faint outlines still show through:

arcs.on("mouseenter", (event, d) => {
    const touches = r => r.source.index === d.index || r.target.index === d.index;
    ribbons.attr("fill-opacity",   r => touches(r) ? 0.85 : 0)   // unconnected → invisible
           .attr("stroke-opacity", r => touches(r) ? 1    : 0);
    arcs.attr("opacity", a => connectedTo(a, d) ? 1 : 0.25);     // dim unrelated arcs
});
arcs.on("mouseleave", () => {
    ribbons.attr("fill-opacity", 0.55).attr("stroke-opacity", 1);
    arcs.attr("opacity", 1);
});

4. When chord works (and when it doesn't)

  • ✅ Few nodes (≲ 50), most pairs have a meaningful weight, ordering matters.
  • ✅ Showing the distribution of flow from one group to others — migrations between countries, transitions between states.
  • ❌ Many nodes — labels become unreadable, ribbons cross into a hairball.
  • ❌ Sparse graphs — most ribbons would be invisible; a node-link or matrix view is clearer.
Takeaway. Chord = matrix bent into a circle. Two D3 generators (d3.chord + d3.ribbon) carry the geometry; you supply ordering and a colour scale, then add a hover handler to tame the visual density. Best on small dense graphs where pair-wise flow is the story.
