The production version. Same rescale-scales handler as §4, plus a clipPath declared in defs and applied to a group that wraps the dots. Anything panned outside the plot rectangle is invisible: the clip is a fixed window, the rescaling moves the data through it. Wheel-zoom, drag, double-click — try every gesture; nothing leaks into the margins. Source tutorial →

What this example shows

The §4 rescale-scales handler stays exactly the same. We add one ingredient on top — a clipPath — and the chart becomes ready to ship: marks no longer leak outside the plot rectangle when the reader pans to an edge.

What is clipPath?

A clip-path is an SVG element that defines a shape used as a visibility mask: any pixel of a target element that falls inside the shape is rendered, and any pixel outside is hidden. The clipped element still exists in the DOM and still receives hover events on its visible parts — it is only the rendered pixels that get cut away.

Three things to remember about it:

  • Declared inside <defs>. The defs element holds reusable SVG content (clip paths, markers, gradients, patterns) that the browser parses but does not paint until something references it.
  • Referenced by id. The consumer says clip-path="url(#myclip)". SVG ids share a global namespace per document, so if a page has multiple charts you must use a unique id per chart — otherwise every url(#clip) resolves to the first one and the other charts look unclipped.
  • Lives in the coordinate system of the element that uses it. Because we attach the clip-path to a group inside plot (already translated by the margins), the clip rectangle's (0, 0) corner is the top-left of the inner plot area.

The full structure in this example:

<svg>
  <defs>
    <clipPath id="clip-10e-04">
      <rect width="innerW" height="innerH"/>   <!-- the visibility window -->
    </clipPath>
  </defs>
  <g transform="translate(margin.left, margin.top)">     <!-- plot -->
    <g class="axis"> ... </g>                            <!-- axes: NOT clipped -->
    <g clip-path="url(#clip-10e-04)">                    <!-- dotsG: CLIPPED -->
      <circle .../> <circle .../> ...
    </g>
  </g>
</svg>

How clipping interacts with zoom

  • The clip rectangle is fixed at innerW × innerH pixels — it does not participate in the zoom transform. Think of it as a window cut into a wall.
  • The rescaled scales move the data through the window. When the reader pans, cx/cy of each circle changes; circles that would land outside the window simply aren't painted.
  • Only the dots are clipped — the axis groups deliberately sit outside dotsG so tick marks and labels remain fully visible even when their rendering position coincides with a "hidden" data point.
  • Reset zoom: the button below calls zoom.transform(svg, d3.zoomIdentity) inside a transition. d3.zoomIdentity is the constant transform with k=1, x=0, y=0 — i.e. "no zoom". Routing the reset through the zoom behaviour (instead of mutating attributes directly) keeps D3's internal state in sync, so the next wheel-scroll continues from the reset position rather than jumping back to wherever the user was.
