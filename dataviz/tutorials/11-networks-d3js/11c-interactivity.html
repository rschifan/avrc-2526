11c — Interactivity: drag, hover, click

A static force layout already shows the shape of a network. Three interactions turn it into something you can read: drag (rearrange by hand), hover (focus on a node and its neighbours), click (pin or filter). All three reuse the fx/fy mechanism from 11b and the data join from L9.

1. Drag with d3.drag

Dragging a node is "pin while the pointer is down": set fx/fy on the dragged datum, follow the pointer, then release. D3 ships a behaviour for the pointer plumbing:

function drag(sim) {
    function started(event, d) {
        if (!event.active) sim.alphaTarget(0.3).restart();
        d.fx = d.x;  d.fy = d.y;
    }
    function dragged(event, d) {
        d.fx = event.x;  d.fy = event.y;
    }
    function ended(event, d) {
        if (!event.active) sim.alphaTarget(0);
        d.fx = null;  d.fy = null;
    }
    return d3.drag()
        .container(() => c.plot.node())   // pan/zoom-aware coords
        .on("start", started).on("drag", dragged).on("end", ended);
}

node.call(drag(sim));

Three things to notice:

  • alphaTarget(0.3).restart() — keeps the simulation simmering while dragging, so the rest of the graph reacts instead of freezing.
  • On start we pin to the current position so the node doesn't snap to the pointer's coordinate system.
  • On end we set fx/fy back to null — the node becomes free again and the simulation settles.

2. Hover to highlight neighbours

The information you usually want from a network view is "what is connected to this?". Hover a node, fade everything that is not a neighbour. The trick is to precompute a neighbour set per node so the hover handler is O(1):

const neighbors = new Map(nodes.map(d => [d.id, new Set([d.id])]));
links.forEach(l => {
    const a = l.source.id ?? l.source;
    const b = l.target.id ?? l.target;
    neighbors.get(a).add(b);
    neighbors.get(b).add(a);
});

node.on("mouseenter", (event, d) => {
    const focus = neighbors.get(d.id);
    node.attr("opacity", n => focus.has(n.id) ? 1 : 0.1);
    link.attr("opacity", l => focus.has(l.source.id) && focus.has(l.target.id) ? 0.9 : 0.05);
});
node.on("mouseleave", () => {
    node.attr("opacity", 1);
    link.attr("opacity", 0.5);
});

The l.source.id ?? l.source idiom handles a small wrinkle: forceLink rewrites every link's source/target string into the actual node object, but only after the simulation has been initialised. Reading .id with a fallback to the raw string lets the same code run before and after that swap.

Conditional labels. Labelling every node in a 187-node network is unreadable. The same hover handler can do double duty: hide all labels by default, then reveal only the focal node's neighbourhood on hover:

const label = wrap.append("g").selectAll("text")
    .data(nodes).join("text")
    .attr("text-anchor", "middle").attr("dy", -8)
    .style("font-size", "11px").style("pointer-events", "none")
    .style("opacity", 0)                                       // hidden by default
    .text(d => d.id.replace(/-/g, " "));

sim.on("tick.label", () => {                                  // keep labels glued
    label.attr("x", d => d.x).attr("y", d => d.y);          // to their nodes
});

node.on("mouseenter", (event, d) => {                         // extend the same
    const focus = neighbors.get(d.id);                         // hover handler
    node.attr("opacity", n => focus.has(n.id) ? 1 : 0.1);
    link.attr("opacity", l =>
        focus.has(l.source.id) && focus.has(l.target.id) ? 0.9 : 0.05);
    label.style("opacity", n => focus.has(n.id) ? 1 : 0);   // reveal neighbours
});
node.on("mouseleave", () => {
    node.attr("opacity", 1);
    link.attr("opacity", 0.5);
    label.style("opacity", 0);
});

The tick.label namespace adds a second tick listener without replacing the one that positions nodes and links — without the suffix, a second sim.on("tick", …) call would overwrite the first.

3. Click to pin / unpin

Click a node and it stays where it is — even when the simulation reheats. Click it again and it releases. Same fx/fy mechanism, no drag involved:

node.on("click", (event, d) => {
    if (d.fx == null) {           // currently free → pin at current position
        d.fx = d.x;
        d.fy = d.y;
        d3.select(event.currentTarget).attr("stroke", "#D55E00").attr("stroke-width", 3);
    } else {                       // already pinned → release
        d.fx = null;
        d.fy = null;
        d3.select(event.currentTarget).attr("stroke", "white").attr("stroke-width", 1);
    }
});

4. Click-to-isolate with an animated transition

For a focused view, replace the click-to-pin with a richer interaction: clicking a node slides it to the left and stacks its neighbours in a vertical column on the right; clicking the empty background animates everything back and the force layout resumes. Three pieces fit together:

  • A three-state machine: idle → transitioning → isolated → transitioning → idle. Clicking another node mid-isolation snaps back to the snapshot and isolates the new node immediately.
  • Snapshot positions before isolating, and on dismiss animate back to that snapshot — this keeps the reset calm and reversible.
  • Resume the simulation gently after the dismiss: sync each node's x, y to the snapshot and call sim.alpha(0.15).restart() so the layout is alive again (drag, hover and the force springs all work) without a violent re-equilibration.
let phase      = "idle";   // 'idle' | 'transitioning' | 'isolated'
let snapshot   = null;
let phaseTimer = null;     // cancellable end-of-transition timer
const DURATION = 800;

function isolate(focal) {
    clearTimeout(phaseTimer);
    phase = "transitioning";
    sim.stop();
    if (!snapshot)
        snapshot = new Map(nodes.map(n => [n.id, { x: n.x, y: n.y }]));

    const others = [...neighbors.get(focal.id)].filter(id => id !== focal.id);

    // Centre the column around innerH/2 regardless of count.
    const PITCH = 24;
    const span  = Math.min(innerH - 60, (others.length - 1) * PITCH);
    const top   = innerH / 2 - span / 2;
    const yOf   = others.length <= 1
        ? () => innerH / 2
        : i  => top + (span * i) / (others.length - 1);

    const dest = new Map();
    dest.set(focal.id, { x: 80, y: innerH / 2 });
    others.forEach((id, i) => dest.set(id, { x: innerW - 220, y: yOf(i) }));

    const destX = r => dest.has(idOf(r)) ? dest.get(idOf(r)).x : r.x;
    const destY = r => dest.has(idOf(r)) ? dest.get(idOf(r)).y : r.y;
    const isActive = l =>
        idOf(l.source) === focal.id || idOf(l.target) === focal.id;

    node.transition().duration(DURATION)
        .attr("cx", n => destX(n)).attr("cy", n => destY(n))
        .attr("r",   n => n.id === focal.id ? 9        : 4)
        .attr("fill", n => n.id === focal.id ? "#D55E00" : "#0072B2")
        .attr("opacity", n =>
            (n.id === focal.id || others.includes(n.id)) ? 1 : 0);
    link.transition().duration(DURATION)
        .attr("x1", l => destX(l.source)).attr("y1", l => destY(l.source))
        .attr("x2", l => destX(l.target)).attr("y2", l => destY(l.target))
        .attr("opacity", l => isActive(l) ? 0.85 : 0);

    // After the move, drop the names into place (focal on top, others to right).
    phaseTimer = setTimeout(() => {
        phase = "isolated";
        renderIsolationLabels(focal, others, dest);
    }, DURATION);
}

function dismiss() {
    clearTimeout(phaseTimer);
    phase = "transitioning";
    clearIsolationLabels();

    node.transition().duration(DURATION)
        .attr("cx", n => snapshot.get(n.id).x)
        .attr("cy", n => snapshot.get(n.id).y)
        .attr("r", 4).attr("fill", "#0072B2").attr("opacity", 1);
    link.transition().duration(DURATION)
        .attr("x1", l => snapshot.get(idOf(l.source)).x)
        .attr("y1", l => snapshot.get(idOf(l.source)).y)
        .attr("x2", l => snapshot.get(idOf(l.target)).x)
        .attr("y2", l => snapshot.get(idOf(l.target)).y)
        .attr("opacity", 0.5);

    phaseTimer = setTimeout(() => {
        nodes.forEach(n => {
            const s = snapshot.get(n.id);
            n.x = s.x; n.y = s.y; n.vx = 0; n.vy = 0;
            n.fx = null; n.fy = null;
        });
        sim.alpha(0.15).restart();              // layout is alive again
        snapshot = null;
        phase    = "idle";
    }, DURATION + 20);
}

// Click on a node always isolates it. If we are mid-isolation, first snap
// every mark back to the snapshot (no animation) so the next isolate has a
// clean starting state — this is what prevents the stuck-state.
node.on("click", (event, d) => {
    event.stopPropagation();
    if (phase !== "idle") snapToSnapshot();
    isolate(d);
});
c.root.on("click", event => {
    if (event.target.tagName === "svg" && phase === "isolated") dismiss();
});

Takeaway. Three small handlers cover the bulk of network interaction: drag sets fx/fy while the pointer is down, hover fades anything outside a precomputed neighbour set, and click either toggles a pin or pulls a node and its neighbours into an animated focus view backed by a position snapshot. Every interaction in L11 sits on this same vocabulary.
