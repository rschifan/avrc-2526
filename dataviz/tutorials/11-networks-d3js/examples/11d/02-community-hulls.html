11d §3 — Community hulls

Colour-by-community gets you most of the way; drawing the community boundary finishes the job. d3.polygonHull returns the convex hull of a set of points; curveCatmullRomClosed turns it into a smooth, closed shape. Source tutorial →

What this example shows

Per simulation tick: group nodes by community, compute the convex hull of each group, render a smooth closed path coloured to match the community. The hull layer sits below the links and nodes (so marks draw on top), and recomputes every tick so it tracks the layout as it relaxes.

  • d3.group(nodes, d => d.community) — returns a Map(community → array of nodes). Same idea as Python's itertools.groupby but doesn't require sorting first.
  • d3.polygonHull(points) — Graham scan / Andrew's monotone chain. Takes an array of [x, y] tuples; returns the convex hull as a clockwise array of points. Needs at least 3 points; returns null for collinear inputs — hence the .filter(Boolean) at the end.
  • d3.line().curve(d3.curveCatmullRomClosed.alpha(0.7)) — the Catmull-Rom curve fits a smooth spline through the hull's vertices and closes it. alpha(0.7) controls how tightly the curve hugs the points: 0 = straight segments, 1 = loose loops. 0.7 is a good middle.
  • Render order matters: hullG is the first child of wrap, so links and nodes paint on top of the hulls. Without the layering, the translucent fills would obscure the marks they're supposed to highlight.
  • Performance note: drawHulls() runs every tick — for 187 nodes / 9 communities this is fine. For very large graphs, throttle to every Nth tick, or only redraw on "end".
  • Toggle visibility — checkbox above the canvas. The hulls display toggles without rebuilding anything: a cheap "show me with vs without" comparison.
