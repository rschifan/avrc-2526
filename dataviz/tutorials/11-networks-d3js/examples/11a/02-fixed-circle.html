11a §3 — Fixed circular layout

A network is two layers of marks bound to two arrays — once positions are known, the L9/L10 data join produces the picture directly. No layout algorithm needed. Source tutorial →

What this example shows

The top 20 characters by betweenness centrality are placed by hand on a circle: x = cx + R · cos(angle), y = cy + R · sin(angle). Edges and nodes are then drawn with the same data-join pattern from L9 / L10.

  • Two joins, two arrays: graph.links → <line> and graph.nodes → <circle>. That is the entire skeleton of a network visualization — every L11 demo from here on adds capabilities on top.
  • An id-to-node map (byId = new Map(top.map(d => [d.id, d]))) lets each link look up its endpoint positions. Links carry string ids; nodes carry positions; the map glues them together.
  • Circular layouts are a hand-coded fallback — useful for ≤ 30 nodes where you want a stable, scannable arrangement. They scale terribly to hundreds of nodes, which is why §4 motivates force layouts.
  • Subgraph filter: we only render links whose both endpoints are in the top-20 set. Otherwise you'd see dangling lines pointing into nothing.
