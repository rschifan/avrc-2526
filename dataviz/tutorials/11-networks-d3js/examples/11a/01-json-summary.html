What this example shows

The interchange between NetworkX and D3 is a single JSON document with two arrays: nodes and links. NetworkX writes it via nx.node_link_data(G), D3 reads it via d3.json(url). Every other example in L11 starts from this same file — datasets/got.json.

  • Each node has an id (any string), plus any number of attributes. Here: community (Louvain label, integer) and bc (betweenness centrality, float in [0, 1]).
  • Each link points to two node ids via source and target, and can carry attributes (here, a co-occurrence weight).
  • d3.json returns a Promise. The graph object is delivered inside .then(graph => …). No special parser, no schema validation — whatever NetworkX wrote becomes a plain JS object.
  • Note: source / target are strings now. Once you pass the links to d3.forceLink (11b), D3 rewrites each one in place with the actual node object. The l.source.id ?? l.source idiom handles both states.
