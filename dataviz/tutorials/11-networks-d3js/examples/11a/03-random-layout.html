11a §4 — Random layout (why we need a force simulation)

The full 187-node graph with positions assigned at random. Same join, same marks, no information conveyed about structure. This is the visual problem that d3.forceSimulation exists to solve. Source tutorial →

What this example shows

At 20 nodes the circular layout from §3 worked: a hand-picked geometric arrangement was readable. At 187 nodes that trick fails — any positions we pick without using the structure of the graph hide the community pattern that's there in the data.

  • The picture is a hairball: edges cross uniformly, no clusters, no hubs. Communities exist (we'll see them in 11d), but a random placement makes them invisible.
  • A deterministic random generator (d3.randomLcg(42)) seeds the placement so the same picture renders on every reload. Useful when comparing what changes between examples.
  • What we want: nodes that share many edges should land near each other; unrelated nodes should drift apart. Derive positions from the graph structure, not from a coin flip.
  • What we'll get in 11b: d3.forceSimulation — links act like springs (pull connected nodes together), nodes repel each other (push them apart), a centring force keeps the result on screen. Run it for ~300 ticks and the graph "relaxes" into a meaningful layout.
