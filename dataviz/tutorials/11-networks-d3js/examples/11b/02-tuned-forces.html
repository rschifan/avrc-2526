11b §3 — Tuned forces (live sliders)

Six sliders, one tick handler. Each input rebuilds the simulation with the new parameters and reheats it — see how each force changes the layout's character. Source tutorial →

What this example shows

The default forces are reasonable but not optimal. In practice you parameterise four (or six) and tune them to the data. The pattern: read slider values → stop the old simulation → build a fresh one with the new parameters → call .alpha(0.6) to reheat.

  • Link distance — resting length of the spring. Larger = airier layout; smaller = tighter clusters.
  • Charge strength — pairwise repulsion (negative). More negative spreads the graph out. Range here is [-300, 0]; default is -30.
  • Collide pad — hard minimum gap between node centres. Stops circles from overlapping. Use a function of r when nodes have different sizes.
  • Alpha decay — how fast the simulation cools. Higher = settles faster (less wiggle, may freeze before untangling). Lower = runs longer (lets communities migrate, but feels sluggish). Default ≈ 0.0228.
  • forceX / forceY strength — per-axis pull toward a target. Set both to 0.05 to gently anchor the centre of mass, larger to elongate the layout.
  • Why rebuild on every input: forceSimulation is stateful. Swapping a single force in place is possible (sim.force("charge", newCharge)), but rebuilding from scratch is simpler and reliably picks up every parameter change.
