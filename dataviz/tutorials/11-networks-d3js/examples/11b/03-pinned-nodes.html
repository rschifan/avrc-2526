11b §5 — Pinning nodes with fx / fy

A node with fx, fy set is fixed: it does not move under any force, and other nodes arrange around it. Setting them back to null releases the node. This same mechanism powers drag (next tutorial). Source tutorial →

What this example shows

Three high-centrality characters are anchored at fixed positions (top centre, lower left, lower right). The rest of the network re-equilibrates around them. Toggle the pins off and the simulation re-runs without anchors — community structure rearranges.

  • Two special properties: each node datum can carry fx (forced x) and fy (forced y). When set, the simulation overwrites d.x with d.fx at the end of every tick. The node still exists in the simulation and still influences neighbours via forceLink and forceManyBody; it just doesn't get pushed around.
  • The release: d.fx = null; d.fy = null; — the simulation stops overriding the position and the node becomes free again.
  • Why reheat after toggling: sim.alpha(0.5).restart() bumps the temperature back up so the layout can adapt to the new constraints. Without it, alpha has decayed to ~0 and the simulation won't move anything.
  • This is the foundation for drag: 11c will wire a pointer's (event.x, event.y) to a node's fx / fy, pinning it as long as the mouse is down, then releasing on mouseup.
  • Use case beyond toy demos: anchor known entities (a focal user, a known cluster head, a fixed origin) and let the rest of the graph arrange in relation to them — useful for comparing layouts across snapshots.
