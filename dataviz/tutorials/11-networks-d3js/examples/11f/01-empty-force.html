11f — Stage 1: Empty force layout

The baseline of the progressive build: d3.forceSimulation with defaults, default radii, a single colour. Every subsequent stage adds exactly one capability on top of this. Source tutorial →

What this stage adds

Nothing. This is the starting point — the L9 data join applied to the L11 dataset and wired to a default force simulation. Everything else in the stage 2 → stage 7 sequence is a delta on this picture.

  • Four default forces: forceLink, forceManyBody, forceCenter, plus boundaryForce to clamp positions to the canvas rectangle every tick. Without that last one the layout drifts off-canvas in either direction.
  • Default radius and colour: every node is the same blue 4px circle. The encodings come in stage 2.
  • Compare with 11b §1: that demo uses fitToCanvas instead — it lets the simulation drift freely and applies a single transform after the simulation cools, to fit the result. Both keep the graph visible; boundaryForce is the easier default because node coordinates stay in canvas space, which makes every later interaction (drag, hover, click) easier to reason about.
  • What this picture tells you: nothing about communities, nothing about centrality. You can see the rough shape — two main clusters connected by bridges — but the data attributes are invisible. stage 2 fixes that.
