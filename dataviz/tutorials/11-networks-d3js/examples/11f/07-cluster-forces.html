11f — Stage 7: Cluster forces (community-aware layout)

A per-community forceX / forceY pulls each node toward its community's centroid. Communities separate during layout instead of being coloured after the fact. The state-of-the-art tuning that pure forceCenter gets wrong. Source tutorial →

What this stage adds (vs the previous stage)

One pair of forces — forceX and forceY with a per-node target derived from its community. Strength 0.08 is gentle enough that link springs and charge still shape the within-community geometry, but strong enough that communities visibly separate. Drop the strength to 0 and you're back to the stage 3 layout; crank to 0.3 and communities become rigid blobs.

  • Target placement: each community gets a target point on a ring inside the canvas. angle = i / nCommunities · 2π, radius = 0.30 · min(W, H). The ring keeps targets equidistant from the centre — no community starts privileged.
  • Per-datum strength: d3.forceX(d => tgt.get(d.community).x). The function form lets each node aim at a different target. This is the modern idiom — a single forceX with a varying target beats N separate forces.
  • Why this is better than hulls: the hull view (11d §3) draws a boundary around whatever the layout happens to produce. Cluster forces produce the boundary by separating communities at simulation time. Less visual machinery, more honest signal.
  • Why forceCenter is missing: forceCenter pins the overall centre of mass — useful for one blob, bad when you want multiple blobs to pull apart. The forceX + forceY pair with the canvas centre as target replaces it cleanly, and per-node x / y targets generalise from there.
  • Tuning hint: if communities overlap visibly, raise the cluster strength or weaken the link force (smaller distance / stronger charge). If they fly apart and miss the canvas, lower the cluster strength or pull the targets inward (smaller radius).
  • Cross-reference: similar trick scales to dozens of communities via the forceInABox plugin (treemap-based packing), or d3-force-cluster (custom Verlet integrator). For under ~10 groups the simple forceX / forceY pair is enough and ships in core d3-force.
