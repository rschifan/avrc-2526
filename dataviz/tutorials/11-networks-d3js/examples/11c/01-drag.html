11c §1 — Drag with d3.drag

Dragging a node is "pin while the pointer is down": set fx / fy on the dragged datum, follow the pointer, then release on mouseup. Same mechanism as 11b §5 — just animated by a pointer. Source tutorial →

What this example shows

d3.drag() is a behaviour object (like d3.zoom) that wires up pointerdown / move / up listeners and emits start / drag / end events. You handle those three events to set, update, and clear fx / fy on the dragged node.

  • On start: d.fx = d.x; d.fy = d.y; — pin at the current position so the node doesn't snap to the pointer's coordinate system. Also sim.alphaTarget(0.3).restart(): keep the simulation simmering while dragging, so the rest of the graph reacts instead of freezing into place.
  • On drag: d.fx = event.x; d.fy = event.y; — follow the pointer. The next tick reads these fixed values and overwrites d.x / d.y; everything else moves under normal force physics.
  • On end: d.fx = null; d.fy = null; — release. Also sim.alphaTarget(0) so the simulation cools back down to rest. The !event.active check guards against multitouch: only the last finger lifting should reset the target.
  • node.call(drag(sim)) applies the behaviour to every circle in the selection — D3 attaches the right listeners under the hood.
  • This is the foundation for click-to-pin (next example) and the shift+click isolate-animation in 11e. Both are variations on "imperatively set fx / fy on the affected node(s)".
