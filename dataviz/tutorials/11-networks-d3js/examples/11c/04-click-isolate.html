11c §4 — Click-to-isolate animation

Click any node → it slides to the left, its neighbours stack on the right, active edges follow, the rest fades. Click anywhere again → everything transitions back to where it was and the force layout resumes. Source tutorial →

What this example shows

Two animations driven by a three-state machine (idle / transitioning / isolated) and a position snapshot taken at click time.

  • Click on a node: snapshot every node's current (x, y), stop the simulation, animate the focal node and its neighbours to the destination layout (left anchor + right column on the right). Clicking another node mid-isolation snaps the marks back to the snapshot and isolates the new node immediately.
  • Click the empty background: animate every mark back to its snapshot position. When the transition ends, sync the simulation's node data to the snapshot and call sim.alpha(0.15).restart() so the layout is alive again — drag, hover and the force springs all work.
  • The destination map: dest = Map(id → {x, y}). The focal id maps to the left anchor; each neighbour id to its slot on the right column. Both the node transition and the link-endpoint transition read from this same map, so edges follow the nodes that move them.
  • Use case: focused view of one node's ego network. The capstone in 11g binds it to shift+click so the plain click can still pin.
