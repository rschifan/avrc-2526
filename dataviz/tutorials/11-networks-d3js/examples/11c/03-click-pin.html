11c §3 — Click to pin / unpin

Click a node and it stays where it is — even when the simulation reheats. Click it again and it releases. Same fx / fy mechanism as 11b §5, but driven by an event rather than a hard-coded list. Source tutorial →

What this example shows

The handler reads the current pin state directly off the datum: d.fx == null means "currently free" → pin at current position. Anything else means "currently pinned" → release. State lives on the data, not in a separate set — which makes it survive reheats, joins, and re-renders for free.

  • Pin at current position, not at event.x: the user clicked a node — they expect it to stay where they clicked it, not snap to the pointer's exact pixel.
  • Visual feedback via d3.select(event.currentTarget): pinned nodes get an orange ring (stroke: #D55E00; stroke-width: 3), free nodes go back to white/1px. The handler only mutates the clicked circle — cheaper than a full re-render.
  • Drag still works: when the pin is held by fx/fy, dragging the node simply overwrites them with new values; releasing the mouse without resetting them keeps the node pinned at its new location. Comment out the drag block to see clicks-only behaviour.
  • Combine with hover for the canonical interaction trio: drag to rearrange, hover to inspect connectivity, click to pin permanently. That's the interaction vocabulary every L11 demo uses.
  • Edge case: event.stopPropagation() is sometimes added to prevent the click from bubbling to a background handler (background clicks reset state in the 11e capstone). Not needed here, but worth remembering for richer apps.
