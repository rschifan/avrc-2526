11c §2 — Hover to highlight neighbours

The information you usually want from a network view is "what is connected to this?". Hover a node; everything that is not a neighbour fades; labels reveal only on the focal node's neighbourhood. Source tutorial →

What this example shows

We precompute a neighbour set per node once at load time and look it up in O(1) during hover.

  • Precomputed neighbour map: new Map(nodes.map(d => [d.id, new Set([d.id])])) seeds each entry with the node itself; a single pass over links adds each endpoint to the other's set. Each entry includes the node itself so the focal node is just another member of the focus set during hover.
  • The l.source.id ?? l.source idiom: before forceLink initialises, links carry raw string ids; after, D3 has rewritten them with the actual node object. Reading .id with a fallback to the raw string works in both states.
  • Tick handler namespacing: sim.on("tick.label", …) attaches a second listener to the same tick event, keyed by the .label suffix. Without the suffix, a second sim.on("tick", …) would replace the first. Same pattern works on mouse events (node.on("mouseenter.label", …)).
  • White stroke under text: paint-order: stroke; stroke: white; stroke-width: 3 gives each label a halo so it stays readable when it crosses dark edges or other nodes.
