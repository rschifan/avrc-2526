How the source is organised

Read top-to-bottom — each section corresponds to a tutorial concept and is named with an upper-case banner comment. Shared helpers live at the top; state variables in one block; rendering, simulation, and interactions in their own sections.

  1. Helpers — small pure functions reused across sections.
  2. Data + scales — JSON load, colour / radius scales, neighbour map.
  3. Canvas + marks — makeCanvas plus the five layers (hulls, links, nodes, labels, tooltips).
  4. State — the few lets the rest of the code mutates.
  5. Rendering functions — drawHulls, applyLabelDisplay, applyFilter. Each one's only job is writing attributes.
  6. Simulation — d3.forceSimulation + tick handler.
  7. Interactions — drag, hover, click (pin / isolate / dismiss), zoom.
  8. UI controls — search box, hulls toggle, community legend, reset.
