DV11 — Networks in D3.js

What you should already know

This lecture composes pieces from earlier classes:

  • Graph terminology, communities, centrality (L7 — networks with NetworkX)
  • SVG primitives, the data join, scales (L9)
  • Joins, transitions, and event handling (L10)

Tutorials

Run them in order — each one builds on the previous.

  1. 11a — From NetworkX to D3
    The node-link JSON format produced by nx.node_link_data; the GoT co-occurrence dataset; drawing nodes as <circle> and links as <line> at fixed positions; why a layout algorithm is needed once positions aren't given.
  2. 11b — Force layouts
    d3.forceSimulation; the four standard forces (forceLink · forceManyBody · forceCollide · forceCenter); the tick handler; alpha / alphaDecay; pinning a node via fx / fy.
  3. 11c — Interactivity: drag, hover, click
    d3.drag with alphaTarget + fx / fy; precomputed neighbour set for hover-to-highlight (with conditional labels); click-to-pin toggle; click-to-isolate animation with a small state machine and position snapshots.
  4. 11d — Visual encoding and community overlays
    scaleOrdinal(schemeTableau10) for community colour; scaleSqrt for centrality-by-area; convex-hull community overlays via d3.polygonHull + curveCatmullRomClosed.
  5. 11e — Capstone
    Full GoT viewer combining everything: drag, hover-fade, click-pin, semantic zoom, search box, community filter (click legend swatch to solo / toggle communities).
  6. 11f — Progressive build
    Seven stages, each adding one capability on top of the previous: empty force layout → encoded base → drag & hover → zoom-aware labels → foreignObject styled labels → perf / scale instrumentation → cluster forces. Stages 1–5 are the visible build; stages 6 and 7 answer the production questions the topic-by-topic tutorials skip — how the layout scales and how community structure can emerge from the layout itself.
  7. 11h — Web Worker simulation
    Moving the force simulation off the main thread. The postMessage protocol; Transferable typed arrays (Float32Array.buffer) so per-tick payloads are O(1) instead of O(n); drag-through-messages. The recipe for keeping the UI responsive while a large layout converges.

Setup

The tutorials are plain HTML files and self-contained — every asset they need (_shared/chart.css, _shared/chart.js, datasets/got.json) is inside this folder. Serve them with a local HTTP server:

cd 11-networks-d3js
python -m http.server 8000
# open http://localhost:8000/

D3 is loaded from the CDN (https://d3js.org/d3.v7.min.js); no installation required.

Navigating each demo. Every L11 demo has pan and zoom on the SVG stage by default: drag the empty background to move the view (handy for seeing nodes that drifted past the visible area), wheel / pinch to zoom in or out. Node drag still works under pan because every drag behaviour declares the plot <g> as its .container(…), which makes the pointer coordinates resolve in the post-transform coordinate space.
