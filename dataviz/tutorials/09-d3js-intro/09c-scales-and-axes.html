09c — Scales, Axes, and Shape Generators

Scales turn raw data values into visual values (pixel positions, radii, colors). Axes are the visible record of that mapping. Shape generators (d3.line, d3.area) consume scales pixel-by-pixel and emit the SVG d string for line and area charts. All three together cover the full data → pixel pipeline.

How this tutorial is organised

Four blocks, in order:

  1. Why scales, and the basic linear scale (§1-§2). What goes wrong without one, then scaleLinear with an interactive domain demo and the .nice() / .clamp() helpers.
  2. Axes and the chart skeleton (§3-§5). d3.axisBottom configurations, the margin convention that gives axis labels room to breathe, and what changes when you go from one scale to two (the inverted y-range).
  3. Picking the right scale type (§6-§7). scaleLog, scaleSqrt, scaleBand, scaleOrdinal — when each one is the right tool, with side-by-side comparisons.
  4. From scales to shapes (§8-§12). d3.line, the .datum() vs .data() distinction, curves as a design choice, d3.area, and a ridgeline composition that reuses one area generator across many groups.

1. Why we need scales at all

Suppose we have nine data points with values from 0 to 50 000:

const data = [120, 480, 2400, 5500, 9800, 17000, 24000, 38000, 49500];

If we map them directly to pixel positions, most fall off the right of the plot area:

A scale fixes this. It is a function that compresses the data range into the pixel range:

const x = d3.scaleLinear()
    .domain([0, 50000])    // input: data values
    .range ([0, innerW]);  // output: pixel positions

x(0);       // → 0
x(25000);   // → innerW / 2
x(50000);   // → innerW

3. Configuring an axis

An axis is a generator: pass it a scale and it draws ticks, tick labels, and the axis line. The same scale renders four different ways below — only the axis configuration changes:

// (1) default
d3.axisBottom(x)

// (2) custom number of ticks
d3.axisBottom(x).ticks(4)

// (3) explicit tick values
d3.axisBottom(x).tickValues([0, 12500, 25000, 37500, 50000])

// (4) custom tick formatting (SI prefixes — 10k, 20k, …)
d3.axisBottom(x).tickFormat(d3.format("~s"))

Other axis variants: d3.axisTop (ticks above the line), d3.axisRight (ticks on the right). All four work the same way.

4. The margin convention

The reason every chart in this tutorial uses the same skeleton: an axis lives at the edge of the plot area. If the plot area starts at the SVG edge, the axis line and tick labels get clipped. The margin convention pads the SVG and puts the marks inside an inner <g> shifted by the margins.

const width  = 700, height = 200;
const margin = { top: 24, right: 30, bottom: 36, left: 50 };
const innerW = width  - margin.left - margin.right;   // 620
const innerH = height - margin.top  - margin.bottom;  // 140

const svg = d3.select("#chart")
    .append("svg")
        .attr("width",  width)
        .attr("height", height)
    .append("g")
        .attr("transform", `translate(${margin.left}, ${margin.top})`);

Below, the same axis rendered two ways:

Without margins — axis line sits at the SVG top edge; tick labels rendered above it (y < 0) are clipped and disappear.

With margins — the inner <g> is shifted by the margins, so axis labels have room and stay readable.

5. Two scales make a 2D plot

A 2D chart needs one scale per axis. Both can be linear, but the y-scale's range is inverted: [innerH, 0] instead of [0, innerH], so larger data values appear higher on the screen.

const y = d3.scaleLinear()
    .domain([0, 100])
    .range ([innerH, 0]);    // ← inverted on purpose

6. When linear is wrong: scaleLog and scaleSqrt

A linear scale gives every numeric step the same pixel space. That is not always what you want:

  • Values spanning orders of magnitude: a linear axis crushes the small values into a thin strip. Use scaleLog.
  • A radius that should encode area: the human eye reads circle area, not radius. Use scaleSqrt so that area = π r² ∝ value.

Same data on a linear axis (top) vs. a log axis (bottom):

const data = [10, 30, 100, 300, 1000, 3000, 10000, 30000];

const xLin = d3.scaleLinear().domain(d3.extent(data)).range([0, innerW]);
const xLog = d3.scaleLog()   .domain(d3.extent(data)).range([0, innerW]);

Same data with r = scaleLinear (top) vs. r = scaleSqrt (bottom):

const values = [1, 4, 9, 16, 25, 36, 49];

// radius proportional to value → area grows quadratically (visually exaggerated)
const rLin  = d3.scaleLinear().domain([0, d3.max(values)]).range([0, 40]);

// radius proportional to √value → area is proportional to value (correct)
const rSqrt = d3.scaleSqrt()  .domain([0, d3.max(values)]).range([0, 40]);

7. Categorical scales: scaleBand and scaleOrdinal

Continuous scales handle numbers. For categories we need a different mapping:

  • scaleBand — discrete categories → equal-width slots. Used for bar-chart x-axes; provides .bandwidth() for the slot width.
  • scaleOrdinal — discrete categories → discrete values (typically colors).
const xBand = d3.scaleBand()
    .domain(["Mon","Tue","Wed","Thu","Fri","Sat","Sun"])
    .range ([0, innerW])
    .padding(0.15);

xBand("Wed");          // → left edge of the "Wed" slot
xBand.bandwidth();     // → slot width minus padding

const color = d3.scaleOrdinal()
    .domain(["A", "B", "C"])
    .range (["#0072B2", "#E69F00", "#009E73"]);

color("B");            // → "#E69F00"

8. From scales to shapes — d3.line

A line chart is a single <path> whose d attribute traces a polyline through every data point. Writing that string by hand for hundreds of points would be tedious; d3.line() is a generator that does it for you, applying scales pixel-by-pixel:

const line = d3.line()
    .x(d => xScale(d.year))         // ← scale from §2 reused here
    .y(d => yScale(d.value));       // ← scale from §5

svg.append("path")
    .datum(timeSeries)              // bind the WHOLE array to ONE element
    .attr("d", line)                 // line(timeSeries) → "M … L … L …"
    .attr("fill", "none")
    .attr("stroke", "#0072B2").attr("stroke-width", 2);

9. .datum() versus .data()

This distinction trips up everybody once.

  • .data(array) binds each row of the array to one element in the selection. Used for scatterplots, bar charts — one mark per row.
  • .datum(value) binds one whole value (typically the entire array) to one element. Used for line charts and area charts — one path that consumes many rows.
// scatterplot: one circle per row
svg.selectAll("circle")
    .data(rows)               // ← data, plural
    .join("circle");

// line chart: one path consuming the whole array
svg.append("path")
    .datum(rows)              // ← datum, singular
    .attr("d", line);

10. Curves are design choices

By default d3.line connects points with straight segments. For smoother shapes, configure a .curve(...):

const line = d3.line()
    .x(d => x(d.year)).y(d => y(d.value))
    .curve(d3.curveMonotoneX);   // ← here

Common curves, each with a distinct visual character:

  • curveLinear — straight segments (default)
  • curveStep, curveStepAfter, curveStepBefore — staircase variants; honest about discrete change
  • curveMonotoneX — smooth, restrained, never overshoots vertically (the safe default for continuous data)
  • curveCatmullRom — smoother, more decorative
  • curveBasis — heavily smoothed; may not pass through points

11. d3.area — the area generator

An area chart is also one <path>, but the path needs both an upper and a lower boundary. d3.area() takes x, y0 (lower edge), and y1 (upper edge):

const area = d3.area()
    .x(d => xScale(d.year))
    .y0(innerH)                  // baseline at bottom of plot
    .y1(d => yScale(d.value));

svg.append("path")
    .datum(timeSeries)
    .attr("d", area)
    .attr("fill", "#0072B2").attr("opacity", 0.3);

Same data, same generator pattern as d3.line. Both accept .curve(...); a common composition is a filled area underneath plus a stroked line on top, both consuming the same array via .datum(...):

Takeaway. One scale per visual channel; pick the type from the shape of the data, not from the chart you want. The margin convention plus an axis generator per scale completes the chart skeleton. d3.line and d3.area are the scale-driven shape generators that turn an array of rows into a single SVG path string — line and area charts emerge by adding two ingredients (a generator + a .datum(array) binding) on top of the scale machinery. The toy datasets so far were hand-typed; 09d introduces d3.csv, the accessor pattern, and the helpers (extent · group · rollup · format) needed to feed real data into all of this.
